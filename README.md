# اللجنة الطبية العسكرية - الواجهة الخلفية (API)

## نظرة عامة

يحتوي هذا المستودع على واجهة برمجة التطبيقات (API) لتطبيق اللجنة الطبية العسكرية. يوفر هذا الـ API الخدمات الخلفية اللازمة للواجهة الأمامية (الموجودة في مستودع `love`)، بما في ذلك إدارة البيانات، المصادقة، والوظائف السحابية (Serverless Functions).

## الميزات الرئيسية

*   **إدارة البيانات:** يتفاعل مع قاعدة بيانات Supabase لإدارة معلومات اللجان الطبية، المرضى، الأطباء، والمواعيد.
*   **وظائف بلا خادم (Serverless Functions):** يستخدم Supabase Edge Functions لتنفيذ منطق الأعمال بكفاءة وقابلية للتوسع.
*   **مصادقة آمنة:** يوفر نقاط نهاية للمصادقة الآمنة للأطباء والإدارة.
*   **نقاط نهاية RESTful:** واجهة برمجة تطبيقات واضحة ومنظمة لسهولة التكامل مع الواجهة الأمامية.

## التقنيات المستخدمة

*   **Node.js:** بيئة تشغيل JavaScript.
*   **Supabase:** منصة مفتوحة المصدر توفر قاعدة بيانات PostgreSQL، مصادقة، وظائف سحابية، وتخزين ملفات.
    *   `@supabase/supabase-js`: مكتبة JavaScript للتفاعل مع خدمات Supabase.
*   **Vitest:** إطار عمل اختبار سريع لوحدات JavaScript/TypeScript.

## الإعداد والتثبيت

لإعداد المشروع محليًا، اتبع الخطوات التالية:

1.  **استنساخ المستودع:**
    ```bash
    git clone https://github.com/Bomussa/love-api.git
    cd love-api
    ```

2.  **تثبيت التبعيات:**
    ```bash
    npm install
    ```

3.  **تكوين المتغيرات البيئية:**
    يتطلب هذا المشروع متغيرات بيئية خاصة بـ Supabase (مثل `SUPABASE_URL`, `SUPABASE_ANON_KEY`). يجب إعدادها في بيئة النشر (مثل Supabase Project Settings) أو في ملف `.env.local` للتطوير المحلي.

4.  **تشغيل خادم التطوير (للاختبار المحلي):**
    ```bash
    npm run dev
    ```
    (ملاحظة: هذا يشغل خادمًا بسيطًا لاختبار الـ API محليًا، ولكن النشر الفعلي يتم عبر Supabase Edge Functions).

## النشر

يتم نشر الواجهة الخلفية كوظائف بلا خادم (Edge Functions) على [Supabase](https://supabase.com/).
